# camera-image-input 要件定義

## プロジェクト説明

カメラで撮影した画像から入力値を取得。血圧計の表示画面を撮影し、収縮期血圧・拡張期血圧・脈拍の数値を自動認識して入力フォームに反映する。

---

## 1 カメラ起動・撮影

### 説明
ユーザーがメイン入力画面からカメラ機能を呼び出し、血圧計を撮影できる。

### 受け入れ基準

- **1.1** WHEN ユーザーがメイン画面のカメラボタンをタップしたとき、アプリはカメラ撮影画面を起動しなければならない。
- **1.2** WHEN アプリがカメラ権限を持っていないとき、アプリはユーザーにカメラ権限を要求しなければならない。
- **1.3** IF ユーザーがカメラ権限を拒否したとき、アプリは権限が必要である旨のメッセージを表示し、手動入力に戻らなければならない。
- **1.4** WHEN ユーザーがシャッターボタンをタップしたとき、アプリは撮影した画像を取得しなければならない。
- **1.5** WHEN ユーザーがカメラ画面をキャンセルしたとき、アプリはメイン入力画面に戻り、入力済みの値を保持しなければならない。

---

## 2 画像からの数値認識（OCR）

### 説明
撮影した画像から血圧値（収縮期・拡張期・脈拍）を自動的に認識する。

### 受け入れ基準

- **2.1** WHEN 撮影が完了したとき、アプリは画像を解析して数値を認識しなければならない。
- **2.2** WHEN 解析中のとき、アプリはローディング表示を行いユーザーに処理中であることを伝えなければならない。
- **2.3** アプリは収縮期血圧（最高血圧）・拡張期血圧（最低血圧）・脈拍の3つの数値を個別に識別しなければならない。
- **2.4** WHEN 数値の認識が成功したとき、アプリは認識した値をメイン入力画面の各フィールドに自動入力しなければならない。

---

## 3 認識値の検証

### 説明
認識した数値が血圧記録として妥当な範囲内であることを確認する。

### 受け入れ基準

- **3.1** IF 認識した収縮期血圧が有効範囲（60〜300 mmHg）外のとき、アプリはその値を無効とみなし認識失敗として扱わなければならない。
- **3.2** IF 認識した拡張期血圧が有効範囲（30〜200 mmHg）外のとき、アプリはその値を無効とみなし認識失敗として扱わなければならない。
- **3.3** IF 認識した脈拍が有効範囲（20〜300 bpm）外のとき、アプリはその値を無効とみなし認識失敗として扱わなければならない。
- **3.4** IF 認識した収縮期血圧が拡張期血圧以下であるとき、アプリはその組み合わせを無効とみなし認識失敗として扱わなければならない。

---

## 4 認識結果の確認・修正

### 説明
ユーザーが認識された値を確認し、必要に応じて修正できる。

### 受け入れ基準

- **4.1** WHEN 認識結果がフォームに反映されたとき、アプリは認識による自動入力であることをユーザーに視覚的に示さなければならない。
- **4.2** WHILE 認識値がフォームに入力されているとき、ユーザーは各フィールドの値を直接編集できなければならない。
- **4.3** アプリは認識結果を上書きして手動で値を入力できるようにしなければならない。
- **4.4** WHEN ユーザーが認識値を修正したとき、アプリは修正された値を登録データとして扱わなければならない。

---

## 5 エラーハンドリング

### 説明
画像認識が失敗した場合や画像が不鮮明な場合の適切な対応。

### 受け入れ基準

- **5.1** IF 画像から数値を認識できなかったとき、アプリは認識失敗のメッセージを表示しなければならない。
- **5.2** IF 認識失敗のとき、アプリはユーザーに再撮影または手動入力の選択肢を提示しなければならない。
- **5.3** IF 画像の明るさや鮮明度が不十分なとき、アプリは撮影条件の改善を促すガイダンスを表示しなければならない。
- **5.4** IF カメラデバイスが利用不可のとき、アプリはカメラが使用できない旨を表示し、手動入力に誘導しなければならない。

---

## 6 UI統合

### 説明
カメラ機能がメイン入力画面に自然に統合され、既存のワークフローを損なわない。

### 受け入れ基準

- **6.1** アプリはメイン入力画面にカメラ起動ボタンを表示しなければならない。
- **6.2** WHEN カメラ機能が利用可能なとき、アプリはカメラボタンを有効状態で表示しなければならない。
- **6.3** アプリはカメラ機能を使用せずに手動入力のみで血圧値を登録できるようにしなければならない（既存機能の維持）。
- **6.4** WHEN 認識処理中のとき、アプリはカメラボタンおよび送信ボタンを無効化しなければならない。
